How Core Chain Works: Mine / Stake / Delegate → Score → Elect Validators → Earn

01

Bitcoin miners delegate hash power

Bitcoin miners include a special OP_RETURN output in their coinbase transactions to signal support for a Core validator — delegating a portion of their hash power without any cost, hardware change, or mining pool disruption.

02

BTC holders time-lock their Bitcoin

BTC holders create a time-locked transaction directly on Bitcoin mainnet using CLTV (CheckLockTimeVerify). This locks BTC for a chosen period and earns CORE token rewards — without BTC ever leaving the Bitcoin blockchain.

03

CORE holders delegate stake

CORE token holders stake their tokens with validators on Core Chain, contributing to the DPoS layer of Satoshi Plus. All three inputs — hash power, BTC time-locks, and CORE stake — combine into a validator score.

04

Top validators elected, rewards distributed

Each round, the top validators by combined Satoshi Plus score are elected to produce blocks. Block rewards and transaction fees are distributed to validators, CORE stakers, and BTC stakers proportionally.

What Is CoreDAO and What Makes Core Chain Different?

Core Chain (governed by CoreDAO) is a Layer 1 blockchain with full EVM compatibility — meaning any Ethereum smart contract or dApp can be deployed on it — secured by a unique consensus model called Satoshi Plus that incorporates Bitcoin's proof-of-work security directly into its validator election process.

Unlike most EVM chains that rely exclusively on their own token for security, Core Chain borrows security from both Bitcoin (via miner hash delegation and BTC time-locking) and its own DPoS layer (via CORE staking). The result is a chain with EVM programmability, Ethereum-compatible tooling, and Bitcoin-backed security — three properties no single chain had previously combined in a production environment.

Satoshi Plus Consensus: How Bitcoin PoW, BTC Staking, and DPoS Work Together

Satoshi Plus is Core Chain's consensus mechanism — a hybrid model that combines three separate security inputs into a single validator scoring and election system. Each validator's score in each round is determined by the sum of these three inputs.

LayerParticipantHow it worksWhat they earn
Bitcoin PoW delegation Bitcoin miners Add Core validator address to coinbase OP_RETURN — signals hash power support with zero cost CORE token rewards from block emissions
Non-custodial BTC staking BTC holders Time-lock BTC on Bitcoin mainnet using CLTV script, point to a Core validator CORE token rewards proportional to BTC locked and duration
Delegated Proof of Stake CORE holders Stake CORE tokens with a chosen validator on Core Chain Block rewards + transaction fees proportional to stake

Each round (1 day), the top 21 validators by combined Satoshi Plus score are elected to produce blocks. The score formula weights all three inputs — meaning a validator with strong miner support, significant BTC time-locked behind them, and large CORE stake delegation will consistently rank highly and earn the most rewards.

Why this design matters: By incorporating Bitcoin's proof-of-work into its consensus, Core Chain makes 51% attacks significantly more expensive than attacking a pure DPoS chain. An attacker would need to not only accumulate CORE stake but also command majority Bitcoin hash power — a far greater resource barrier.

Non-Custodial BTC Staking on Core Chain: How to Earn CORE with Your Bitcoin

Core Chain's BTC staking is fundamentally different from every other "BTC yield" product in crypto. There is no bridge, no wrapping, no custodian, and no smart contract on Core Chain that holds your BTC. The Bitcoin stays on the Bitcoin network — secured by Bitcoin's own infrastructure — under a time-lock you control.

How the time-lock works

You create a Bitcoin transaction that sends BTC to yourself but locks it with CLTV (CheckLockTimeVerify) — a native Bitcoin script opcode that prevents the output from being spent until a specified block height or timestamp. The script includes your chosen Core validator address in the metadata.

CLTV scriptBitcoin-nativeSelf-custodied

How rewards are generated

Core Chain's protocol reads the time-lock metadata on Bitcoin and attributes BTC staking weight to the indicated validator. CORE token rewards are minted on Core Chain and sent to the reward address you specified in the time-lock transaction — your Bitcoin wallet address or any Core Chain address you designate.

CORE rewardsOn Core ChainProportional to BTC × time

BTC staking step-by-step

  1. Choose a validator — research validators by their Satoshi Plus score, commission rate, and track record in the official Core Chain staking interface.
  2. Choose a lock duration — longer locks typically earn higher reward rates. Minimum lock duration applies; check current parameters in the official app.
  3. Create the time-lock transaction — use the official Core staking interface which builds the CLTV script correctly. Do not manually construct this transaction unless you are technically proficient with Bitcoin scripts.
  4. Broadcast and confirm on Bitcoin — send the transaction on Bitcoin mainnet and wait for the required confirmations.
  5. Claim CORE rewards on Core Chain — once the staking is recognised, CORE rewards accrue and can be claimed from the Core Chain staking dashboard.
  6. Await lock expiry to retrieve BTC — your BTC becomes spendable again after the lock height/timestamp passes. No action needed from Core Chain to unlock — Bitcoin's own script handles expiry.
Critical: Once a time-lock transaction is broadcast to Bitcoin, it cannot be reversed. BTC is locked until the specified block height — not retrievable earlier under any circumstances. Choose lock duration carefully and only lock BTC you can afford to be illiquid for that period.

Validators on Core Chain: Roles, Requirements, and How the Election Works

Core Chain elects 21 active validators per round (each round is approximately 1 day) from a larger set of registered validators, based on their combined Satoshi Plus score.

Validator requirements

Register with a minimum CORE self-stake, maintain a full Core Chain node with high uptime, and accept delegations from CORE stakers and BTC stakers. Validators set their own commission rates on delegator rewards.

Minimum self-stakeFull nodeHigh uptime

Validator score composition

Score = weighted sum of: delegated Bitcoin miner hash power + BTC time-locked in their favour + CORE tokens staked with them. The formula balances all three Satoshi Plus layers — no single layer dominates.

Hash powerBTC lockedCORE staked
Validator selection for delegators: Choose validators with a strong and consistent score across all three Satoshi Plus layers, low commission, high uptime, and transparent operations. A validator that relies on only one of the three layers is more vulnerable to score fluctuations than one with balanced multi-layer support.

coreBTC: Wrapped Bitcoin on Core Chain — How It Works and When to Use It

coreBTC is a decentralised, trust-minimised wrapped BTC token on Core Chain — a way to bring BTC liquidity into Core Chain's DeFi ecosystem while keeping the wrapping process as decentralised as possible.

How coreBTC is minted

Users send BTC to a decentralised locker network. Lockers are over-collateralised with CORE tokens as insurance against theft or failure. Once BTC is verified in a locker, equivalent coreBTC is minted on Core Chain and usable in DeFi protocols.

Best Practices for BTC Stakers and CORE Holders on Core Chain

For BTC stakers

  • Only use the official Core staking interface to construct time-lock transactions — incorrect CLTV script construction can result in permanently inaccessible BTC. This is irreversible and cannot be rescued.
  • Choose lock duration conservatively at first — test with a small BTC amount and a short lock duration to verify the full workflow (lock, reward claim, unlock) before committing larger amounts.
  • Never lock BTC you may need before expiry — emergencies, market volatility, or changing plans don't override Bitcoin's CLTV script. The lock is absolute.
  • Verify the reward address carefully — CORE rewards go to the address specified in your time-lock transaction. Ensure this is a Core Chain address you control and can access.
  • Research your chosen validator before locking BTC behind them — a poorly performing validator earns fewer rewards for all BTC stakers delegating to them.

For CORE stakers and delegators

  • Diversify across multiple validators for significant CORE positions — validator downtime or slashing events affect all delegators of that validator.
  • Monitor validator score health — a validator whose miner hash delegation or BTC time-lock support drops significantly will fall in the election ranking and may exit the active set, pausing your rewards.
  • Understand the unbonding period before staking CORE — staked CORE has a withdrawal delay. Do not stake funds needed in the near term.
  • Bookmark the official Core Chain staking URL — phishing sites targeting BTC stakers are dangerous because BTC time-lock errors are irreversible. Verify the domain every session.

Troubleshooting CoreDAO: BTC Staking, Reward Claims, and Validator Issues

"My BTC time-lock transaction was confirmed but I'm not seeing CORE rewards"

  • Core Chain needs to recognise the Bitcoin transaction — this requires a minimum number of Bitcoin confirmations. Allow sufficient time (typically 10–20 Bitcoin confirmations) before rewards begin accruing.
  • Verify the time-lock transaction was correctly formatted — specifically that the Core validator address and reward address are embedded correctly. Use the official Core staking dashboard to check recognition status.
  • CORE rewards accrue over rounds and must be manually claimed via the Core Chain staking interface — they don't automatically transfer to your wallet.

"My chosen validator dropped out of the active set"

  • A validator no longer in the active 21 produces no blocks and generates no rewards for its delegators. Re-delegate your CORE stake to an active validator to resume earning.
  • For BTC time-lock staking, you cannot re-delegate mid-lock — the validator choice is embedded in the Bitcoin transaction. Plan validator selection carefully before locking.

"I can't claim CORE rewards from BTC staking"

  • CORE reward claims require a small CORE gas fee on Core Chain. Ensure your reward address (Core Chain address) has a small CORE balance for gas before attempting to claim.
  • If your reward address is a Bitcoin address rather than a Core Chain address, verify the claiming process in the official documentation — the flow differs between address types.
Bitcoin transaction finality: Any issue with a BTC time-lock transaction is governed by Bitcoin's rules — not Core Chain's. Always verify Bitcoin transaction status on a Bitcoin block explorer (Mempool.space, Blockstream.info) before contacting Core support. The Bitcoin-side confirmation is the first dependency in the entire flow.
